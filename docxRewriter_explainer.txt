docxRewriter.js — Team Explainer
=================================

WHAT PROBLEM DOES THIS FILE SOLVE?
------------------------------------
A .docx file is not a simple text file. Under the hood it's a ZIP archive
containing XML files. The content lives in word/document.xml, wrapped in a
deeply nested structure of tags like <w:p> (paragraph), <w:r> (run = a chunk
of text with a specific style), and <w:t> (the actual text).

When AI rewrites or generates text, it gives us back plain strings. This
file's job is to be the bridge between plain AI text and a properly formatted
Word document.


THE TWO EXPORTED FUNCTIONS
---------------------------

1. extractParagraphs(arrayBuffer) — "Read the doc"

What it does:
  Takes the raw bytes of a .docx file and returns a clean, flat list of all
  meaningful paragraphs.

How it works, step by step:
  1. Unzips the .docx (using JSZip) and reads word/document.xml.
  2. Parses the XML and finds every <w:p> (paragraph) element.
  3. Skips empty paragraphs.
  4. For each paragraph it extracts:
     - index   : its position in the document (critical for writing back later)
     - text    : the full visible text
     - style   : e.g. Heading1, Normal
     - section : which heading it falls under (tracks as it walks the doc)
     - runs    : individual text chunks with their formatting (bold, italic, underline)
     - inTable : whether this paragraph lives inside a table cell

Why `index` matters:
  The paragraph's position in the XML array is the key we use later to say
  "replace paragraph #7 with this new text." Without it we'd have no reliable
  way to map AI output back to the right place.


2. rebuildDocx(arrayBuffer, rewrittenParagraphs) — "Write the doc"

What it does:
  Takes the original .docx bytes plus a list of AI-generated changes, applies
  them to the XML, and returns a new valid .docx as bytes.

The input list can contain three types of operations:

  Operation          | How it's identified          | What it does
  -------------------|------------------------------|------------------------------------
  Update             | has `index`                  | Rewrites an existing paragraph in-place
  Insert             | has `insertBefore/After`     | Adds a new paragraph near a reference
  Append to table    | has `appendToTable`          | Adds new rows to an existing table

The core editing strategy (and why it's clever):
  Instead of rebuilding paragraphs from scratch (which would lose all the Word
  formatting — fonts, spacing, borders, etc.), the code takes a
  "keep the skeleton, swap the meat" approach:

  - It keeps the FIRST run (<w:r>) of a paragraph — this run carries all the
    character formatting (<w:rPr>).
  - It sets that run's text (<w:t>) to the new AI-generated text.
  - It deletes all other runs so no old text bleeds through.

  This means the output document looks like the original — same fonts, same
  styles — just with new words.

For insertions:
  The code clones an existing paragraph with the right style (e.g. Heading2)
  as a template, sets the new text on it, and splices it into the right spot
  in the XML tree.

For new tables:
  It finds the nearest existing table to use as a visual template, clones it,
  strips all its rows, then rebuilds rows from the AI's data — preserving
  column widths and cell formatting from the original.

For appending rows to existing tables:
  It finds the table that contains a reference paragraph (by walking up the
  XML tree), uses the last existing row as a formatting template, and appends
  the new rows to the end of that table.

Finally:
  The updated XML is serialized back to a string, stuffed back into the ZIP
  under word/document.xml, and the whole ZIP is returned as an ArrayBuffer —
  a valid .docx ready to download.


THE BIG PICTURE
----------------

  .docx file (bytes)
        |
        v
   extractParagraphs()
        |  unzip -> parse XML -> walk paragraphs
        v
   Clean list of paragraphs  -->  sent to AI (Claude, etc.)
                                          |
                                  AI returns rewritten text
                                          |
        +---------------------------------+
        v
   rebuildDocx()
        |  unzip -> patch XML -> rezip
        v
   New .docx file (bytes)  -->  user downloads it


WHAT THIS POC PROVES
---------------------
- You can surgically edit a Word document's content from a browser with no
  server, no Word install, just JavaScript.
- AI-generated text can be injected back while fully preserving the original
  document's visual formatting.
- It handles not just simple text replacement but also new paragraphs, new
  tables, and new rows — giving the AI real editorial power over document
  structure.
